What is GIT?

GIT is a system of free distributed version control and open source, designed to handle everything from small to large projects with speed and efficiency, created in 2005 by Linus Torvalds. Wikipedia

The GIT is an optimal tool for work, and one of the main requirements for that are, or wish to join the area of development.

What are the advantages?

Imagine being able to find changes in a project when they were made and who made ?, Imagine you can CTRL + Z throughout your project from the beginning, because that is what the GIT does.

The main advantage of git against other version control systems is their velociade and ability to work offiline.

GIT is currently one of the most discussed topics and is no shortage of tips and commands about it, therefore I will present only the most basic.

Some commands

git add <arquivo> = add file
git commit -s -m "[ACAO] Seu comentario" = Creates a committee
git fetch = upgrading your base
git push = sends file to the repository
git status = shows the status
git checkout = restores the file to the version served
git diff = shows changes
git rebase --continue = when any change in the first rebase
git stash = creates a temporary branch
git blame <arquivo> = see who made the change
git reset --soft HEAD~1 = Resets the last committee

Graphical User Interfaces

Gitk --all = shows graphical area (sudo apt-get install gitk)
